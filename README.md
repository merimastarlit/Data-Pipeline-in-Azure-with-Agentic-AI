# Python 200 Homework Portfolio

This repository is a hands-on portfolio of Python projects that demonstrates end-to-end data work, machine learning, automation, and AI application development. Across these assignments, I built practical solutions that move from raw data to insights, predictions, and workflow automation — the kind of work that is highly relevant for data engineering, analytics, and applied AI roles.

What makes this repository compelling is not just the variety of topics, but the way each project applies real-world engineering principles: data cleaning, statistical analysis, model evaluation, API integration, LLM-powered workflows, cloud storage, and orchestration. These projects reflect a strong foundation in Python programming and a growing ability to build useful tools rather than simply complete isolated exercises.

## What this repository showcases

- Data analysis and visualization using Python, pandas, NumPy, Matplotlib, Seaborn, and SciPy
- Machine learning workflows including regression, classification, feature engineering, and model evaluation
- AI-assisted applications using OpenAI APIs, prompt design, moderation, and language model workflows
- Retrieval-augmented generation (RAG) and semantic search with LlamaIndex and document indexing
- Workflow automation and ETL pipelines using Prefect and Azure Blob Storage
- Cloud-aware thinking through Azure cost analysis and storage-based data pipelines

## Project highlights

### 1. Data analysis and statistical insight
Projects in the early assignments focus on exploring structured data, cleaning it, and turning it into meaningful analysis. These include:
- A global happiness dataset analysis with descriptive statistics, visualizations, and hypothesis testing
- A student performance study that explores what factors influence academic outcomes and builds regression models
- A spam email classification project that compares multiple machine learning models and evaluates real-world classification performance

These exercises demonstrate strong fundamentals in data wrangling, experimentation, and communicating findings clearly.

### 2. Machine learning and predictive modeling
Several projects go beyond basic analysis into model development and evaluation. They show the ability to:
- Prepare and transform data for training
- Build and compare regression and classification models
- Interpret model performance using metrics like RMSE, accuracy, precision, recall, and R-squared
- Recognize limitations and communicate model quality honestly

This is especially valuable for employers because it shows both technical execution and critical thinking around model reliability.

### 3. AI and LLM-powered applications
The later projects explore modern AI workflows, including:
- A job application assistant that rewrites resume bullets, drafts cover letter openings, and applies moderation checks
- A document Q&A system that answers questions from real source documents using retrieval-based techniques
- An AI agent that can analyze a dataset using tools rather than relying on a single hard-coded script

These projects demonstrate practical experience with prompt engineering, tool use, retrieval-based AI systems, and the balance between automation and accuracy.

### 4. Data pipelines and cloud integration
The repository also includes cloud-oriented work that reflects production-style thinking:
- A weather data extraction pipeline that stores raw data in Azure Blob Storage
- An LLM-based weather classification workflow that enriches data records
- An orchestrated ETL pipeline using Prefect to automate the flow from extraction to loading
- A simple Azure cost analysis example showing familiarity with cloud resource planning and cost awareness

These assignments highlight an important modern skill set: building workflows that can scale, be monitored, and integrate with external services.

## Getting started

Install the required dependencies:

```bash
pip install -r requirements.txt
```

Then run individual assignments as needed, for example:

```bash
python assignments_06/project_06.py
```

Some projects require API keys or Azure credentials, so those should be configured in your environment before execution.
